# AI Chat Log → Zenn Article Pipeline

**Purpose:** AI（Gemini/ChatGPT等）との対話ログを元に、Zenn記事のドラフトを作成するワークフロー。

---

## When to Use

- ユーザーが「〇〇の経過を記事にしたい」と言った時
- AI対話ログやスクリーンショットを元にした記事作成時
- 開発中の悪戦苦闘・試行錯誤をコンテンツ化する時

---

## 5ステップ・パイプライン

### Step 1: コンテキスト収集（並列実行）

記事の材料を集める。以下を並列で実行する。

1. **既存記事のスタイル確認** — `articles/*.md` のフロントマター・文体・構成パターンを読む
2. **技術的痕跡の収集** — 設定ファイル、git log、ファイルのタイムスタンプなど、ストーリーの裏付けとなるデータ
3. **執筆ルールの確認** — `CLAUDE.md` のContent Standards、Tone and Style セクション

```
並列で実行:
├─ Glob: articles/*.md → 既存記事のフロントマター確認
├─ Read: CLAUDE.md → 執筆ルール確認
└─ Glob/Read: 関連する設定ファイル・ログ → 技術的痕跡
```

### Step 2: ユーザーに不足情報を質問

AIからは見えないストーリーの「人間側」を聞き出す。AskUserQuestion で以下を聞く。

| 質問 | 目的 |
|:---|:---|
| 動機（なぜやろうと思ったか） | 記事の導入部に使う |
| 苦労ポイント（何にハマったか） | 記事の山場になる |
| 時系列（ざっくりでOK） | ストーリーラインの骨格 |

ユーザーがAI対話ログを提供した場合、ログからストーリーを再構成できるため質問は最小限でよい。

### Step 3: 生ログを `_context/` に保存

```
articles/
├─ _context/                    # Zenn公開対象外
│   └─ {slug}-{source}-log.md  # 例: cursor-to-zed-gemini-log.md
└─ {slug}.md                   # 記事本体
```

- AI対話ログは `articles/_context/` に原文保存する
- ファイル名: `{記事スラッグ}-{ソース名}-log.md`
- 他AIへの入力素材として再利用可能にする
- `_context/` は Zenn の公開対象外（`articles/` 直下の `.md` のみ公開される）

### Step 4: ドラフト作成

以下のルールに従って記事を書く。

#### フロントマター
```markdown
---
title: "タイトル（50-60文字）"
emoji: "⚡"
type: "tech"
topics: ["関連タグ1", "関連タグ2"]
published: false
---
```

#### 構成パターン（体験記事向け）
```
1. 導入 — 何が起きたかの一行サマリ
2. 前提 — 読者が知るべきコンテキスト
3. きっかけ — なぜやろうと思ったか
4. 壁1, 壁2, ... — ハマったポイント（時系列）
5. 転機 — 最適解の発見
6. 結果 — Before/After, 設定の全貌など
7. 教訓 — 学んだこと、汎用的な知見
8. まとめ — 一言で結論
```

#### トーンの注意点（CLAUDE.md準拠）
- **No AI slop** — 「素晴らしい」「革命的」「シームレス」禁止
- **正直に** — 失敗や迷走も書く。成功だけの記事は信用されない
- **事実ベース** — 感想より具体的なデータ（設定値、タイムスタンプ、比較表）

### Step 5: TODO/ロードマップ連携

記事の残作業を追跡可能にする。

1. **`zenn-content/TODO.md`** に記事の状態と残作業を追記
2. **ワークスペースのロードマップ**（`docs/architecture/07-roadmap.md`）にも反映

```markdown
# TODO.md への追記例
| 状態 | スラッグ | タイトル | 備考 |
|:---:|:---|:---|:---|
| 📝 | `new-article-slug` | 記事タイトル | ドラフト済。ログ _context/ に保存済 |
```

## チェックリスト

記事公開前の確認項目（CLAUDE.md + このスキル固有）:

- [ ] `_context/` に生ログが保存されている
- [ ] 既存記事とトーンが統一されている
- [ ] AI slop がない（Grep で「素晴らしい」「革命的」等を検索）
- [ ] 個人情報が匿名化されている（ファイルパス、API キー等）
- [ ] スクリーンショットが必要なら `images/` に配置
- [ ] `TODO.md` に記事の状態が反映されている
- [ ] editor エージェントでレビュー済み
- [ ] `published: true` に変更
